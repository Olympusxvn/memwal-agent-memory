# ADR-013 — Monorepo layout, package boundaries, and naming

## Status

Accepted (Phase 0 foundation).

## Context

MemWal++ is a **pnpm + Turborepo** TypeScript monorepo targeting **Sui Overflow 2026** (Walrus + agentic web). Multiple surfaces (dashboard, agents, CLI) share Move bindings, MemWal wrappers, and local memory — without a single ADR, package boundaries drift and Cursor/agents import across layers incorrectly.

## Decision

### 1. Workspace topology

- **Root:** orchestration only — `package.json` scripts delegate to **Turbo**; no application code at root.
- **Apps (`apps/*`):** deployable or runnable entrypoints — may depend on `packages/*`; **must not** be imported by `packages/*`.
- **Packages (`packages/*`):** reusable libraries — depend inward: `shared` / `ui` at the bottom of the DAG; `core` orchestrates; I/O adapters (`memwal-client`, `local-memory`) sit at the edges.

### 2. Package responsibilities (hard boundaries)

| Package | Owns | Must not |
|---------|------|----------|
| `@memwalpp/shared` | Pure types, constants, zod/schemas without I/O | Import `memwal-client`, Move, React, Node fs |
| `@memwalpp/ui` | Presentational React; design tokens | Business rules, chain clients, secrets |
| `@memwalpp/local-memory` | SQLite / vector / local quality scoring | Sui SDK in hot paths unless explicitly scoped for a future adapter |
| `@memwalpp/memwal-client` | MemWal SDK facade, hooks contract, env config | Dashboard routes, agent business logic |
| `@memwalpp/core` | Domain orchestration, use-cases shared by CLI and agents | Next.js `app/` code |
| `packages/sui-contracts` | Move sources, `Move.toml`, on-chain tests | TypeScript (except optional generated clients elsewhere) |
| `apps/dashboard` | Next.js UX, dApp Kit, wallet | Core domain duplication — call `core` or API layers |
| `apps/agent-swarm` | OpenClaw/NemoClaw skills, swarm wiring | Inline MemWal SDK — use `memwal-client` |
| `apps/cli` | Demos, ops commands | Heavy UI |

**Dependency rule:** `packages/shared` → **zero** internal workspace deps. `packages/ui` may depend on `shared` only. `core` may depend on `shared`, `memwal-client`, `local-memory` (not the reverse).

### 3. Naming conventions

- **Workspace npm scope:** `@memwalpp/<name>` aligned with directory `packages/<name>/`.
- **Directories:** `kebab-case` (`memwal-client`, `agent-swarm`, `sui-contracts`).
- **TypeScript modules:** `*.ts` / `*.tsx`; ESM in packages with `"type": "module"` where already established; import paths use **`.js` extensions** in emitted ESM if required by package `tsconfig` (match existing packages).
- **Move modules:** `snake_case.move` under `packages/sui-contracts/sources/`.
- **Env vars:** `SCREAMING_SNAKE`; MemWal-specific prefix `MEMWAL_*`, chain `SUI_*` / `NEXT_PUBLIC_*` for browser-safe IDs only.

### 4. Documentation layout

- **ADRs:** `docs/decisions/ADR-NNN.md` (this file series).
- **OpenSpec-style feature specs:** `docs/specs/*.md` (canonical); optional `openspec/` reserved for future extracted proposals — until then, do not require a duplicate tree.
- **Architecture narrative:** `docs/ARCHITECTURE.md` (human + Cursor); diagrams under `docs/diagrams/`.

### 5. Consequences

- **Positive:** Clear import lint mental model; agents in Cursor follow one table.
- **Negative:** Occasional duplication of DTOs between Move and TS — mitigate with `shared` types and generated IDs documented in specs, not ad-hoc `any`.
