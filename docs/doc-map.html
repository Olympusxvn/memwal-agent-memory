What this submission is

Read this first (~60 seconds). Full narrative: SUMMARY.md.

Project name MemWal Agent Memory (repo: memwal-agent-memory). Legacy aliases MemWal++ / @memwalpp/* refer to the same codebase.
Official stack Built on Walrus Memory (MemWal) — we wrap @mysten-incubation/memwal, we do not fork it. Relayer handles Seal encryption and embeddings.
Our extension Hybrid memory (local SQLite + redact/quality gates → promote to Walrus) · MCP server for any agent · Mainnet Move marketplace and bounties tied to walrus_blob_id.
Workshop kit Aligned with the official Walrus Memory Workshop curriculum. Judges verify this repository, not the external workshop kit.
North-star demo Bounty posted → hunter recalls context → improves memory → promotes to Walrus → submit_fulfillment(blob_id) on Sui.

Five-minute verification

No wallet, no MemWal API keys, no Sui CLI required. Exit code 0 on every command is the pass bar.

git clone https://github.com/Olympusxvn/memwal-agent-memory.git cd memwal-agent-memory pnpm install && pnpm mcp:build && pnpm mcp:e2e && pnpm agent:demo && pnpm agent:bounty-hunt
CommandWhat exit 0 provesPass
pnpm mcp:e2e MCP stdio server works; memory tools (remember, recall, sync) execute; chain tools return chain_not_configured without keys (expected). Required
pnpm agent:demo Hybrid agent hooks run end-to-end: local store → quality gate → sync narrative with colored [1/N] steps. Required
pnpm agent:bounty-hunt Poster + Hunter agents; recall injects context; bounty fulfillment path references Walrus blob semantics. Required

Judge check: promoted memories pass redaction + quality gate before remember() reaches the MemWal relayer. Success sets walrusBlobId on the local record.

        Sync->>Durable: remember(record)
        Durable->>Relayer: remember(text)
        Relayer->>Walrus: upload encrypted blob
        Walrus-->>Relayer: blobId
        Relayer-->>Durable: jobId and blobId
        Sync->>Local: synced=true, walrusBlobId set
    else gate fails
        Sync->>Local: stays pending
    end

Slide 1 — Title

MemWal Agent Memory

Hybrid verifiable memory for autonomous agents
Walrus Track · Sui Overflow 2026

Slide 2 — Problem

Why this exists

  • Agents lose context across sessions and tools
  • Opaque cloud memory cannot be verified or traded
  • Network latency should not block every recall
Slide 3 — Solution

Three-layer hybrid

  • Local — SQLite, quality scoring, PII redaction (fast, private)
  • MemWal + Walrus — encrypted durable blobs, semantic recall, walrusBlobId
  • Sui Move — marketplace, bounty escrow, royalties on mainnet
Slide 4 — Official stack

Built on Walrus Memory

  • Wraps @mysten-incubation/memwal — not a fork
  • We add hybrid gates, MCP access, mainnet marketplace
  • Judges verify this repo — workshop kit is optional context
